## 1. 서비스 개요

### 1.1 제품 정의

AI 출판 지원 툴은 국내외 POD(Print-On-Demand) 플랫폼에 책을 출판하려는 개인 작가가 기획·집필·디자인·글로벌 출판까지 하나의 툴 안에서 완료할 수 있도록 돕는 AI 기반 SaaS 서비스다.

### 1.2 타겟 사용자

- **1차 타겟** 국내 POD 플랫폼(부크크·교보문고 POD) 이용 예비 작가
- **2차 타겟** 이미 원고가 있으나 출판 과정을 모르는 일반인
- **3차 타겟** Amazon KDP로 글로벌 출판을 원하는 작가

### 1.3 핵심 가치 제안

| 가치 | 설명 |
|------|------|
| 원스톱 | 기획 → 집필 → 교정 → 디자인 → 출판 제출 파일까지 한 화면 |
| AI 협업 | 자연어 대화로 자료 검색·본문 생성·교정 지시, 결과는 작가 승인 후 원고 반영 |
| 규격 보장 | 각 플랫폼 POD 규격에 맞는 워드 파일 자동 생성 및 검증 |
| 데이터 주권 | 원고 저작권은 작가 귀속, AI 학습 미사용, 언제든 DOCX·PDF 내보내기 |

### 1.4 개발 단계

| 단계 | 모듈 | 핵심 기능 | 목표 기간 |
|------|------|-----------|-----------|
| Phase 1 | 모듈 1: AI 집필 지원 | 에디터·자료검색·POD 폼 출력 | 0~4개월 |
| Phase 2 | 모듈 2: 셀링 페이지 | AI 카피·캔바 연동·플랫폼 규격 | 4~8개월 |
| Phase 3 | 모듈 3: KDP 글로벌 | 번역·EPUB·KDP 제출 패키지 | 8개월~ |

| 영역 | 기술 / 서비스 | 선택 이유 |
|------|---------------|-----------|
| 프론트엔드 | Next.js 14 (App Router) | SSR·파일 기반 라우팅·Vercel 원클릭 배포 |

| 파일 스토리지 | Supabase Storage | 원고 첨부 이미지, 생성된 DOCX·PDF 저장 |

| HWP 변환 | hwp.js / LibreOffice CLI | 한국 작가 HWP 파일 업로드 지원. 서버사이드 텍스트 추출 |
| DOCX 생성 | docx (Node.js) | 서버사이드 워드 파일 프로그래매틱 생성 |
| EPUB 생성 | TipTap JSON → HTML → EPUB | pandoc 한글 폰트 깨짐 이슈 회피. 직변환으로 서식 손실 최소화 |
| 장기 실행 작업 | Inngest (백그라운드 잡) | Vercel 60초 타임아웃 우회. EPUB 생성·번역·DOCX 생성 처리 |
| 이메일 발송 | Resend | 트랜잭셔널 이메일. 무료 3,000건/월. 4종 템플릿 |
| Rate Limiting | Upstash Redis + Ratelimit | IP·사용자 기반 분당 호출 제한. Vercel Edge와 궁합 최적 |
| 디자인 연동 | Canva Connect API (심사 중) | 승인 전: AI 텍스트 + 캔바 딥링크로 대체 |
| 배포 | Vercel | Next.js 최적화, 환경변수 관리, 엣지 함수 |
| 모니터링 | Vercel Analytics + Sentry | API 비용·에러·사용량 실시간 추적 |

> **한글 폰트 임베딩** 나눔고딕·Noto Sans KR OTF 파일을 레포에 포함. DOCX 생성 시 자동 임베딩. 클라우드 환경 무관 동작 보장.

> Supabase PostgreSQL 기준. 모든 테이블은 RLS(Row Level Security) 활성화 — 본인 데이터만 접근 가능.

| email | text UNIQUE | 로그인 이메일 |

| plan | enum | `free` \| `basic` \| `pro` |

#### diagnostics — 원고 진단

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| user_id | uuid FK (nullable) | 비회원은 null, 가입 후 연결 |
| session_token | text | 비회원 식별 임시 토큰 (브라우저 쿠키 저장) |
| file_name | text | 업로드 파일명 |
| file_storage_path | text | Supabase Storage 경로 |
| word_count | int | 추출된 글자 수 |

| claimed_at | timestamptz (nullable) | 비회원→회원 전환 후 연결된 시각 |
| created_at | timestamptz | |

> **업로드 지원 형식** DOCX (mammoth) / PDF (pdf-parse) / TXT / HWP (hwp.js 또는 LibreOffice CLI). HWP는 서버사이드 변환 후 텍스트 추출.  
> **플랜별 최대 파일 크기** 무료 100MB / 베이직 200MB / 프로 400MB. 초과 시 "플랜을 업그레이드하면 더 큰 파일을 진단할 수 있습니다" 안내.  
> **비회원 파일 보관** 7일 후 Supabase Storage 자동 삭제. 회원 연결 시 30일 보관. 직접 삭제 요청 즉시 삭제.  
> **비회원→회원 연결 로직** 가입 완료 시 session_token으로 diagnostics 조회 → user_id 업데이트 + claimed_at 기록 → 온보딩 화면에 "진단 결과가 계정에 연결됐습니다" 안내.

## 4. 모듈 1 — AI 집필 지원

### 4.1 2분할 집필 에디터

#### 화면 구성

- **좌측 (60%)** TipTap 리치 텍스트 에디터 — 챕터 선택, 본문 입력, 단어 수 카운터
- **우측 (40%)** AI 챗봇 패널 — 대화 히스토리, 입력창, 기능 버튼 (자료검색·교정·목차기획)
- **상단 바** 프로젝트명, 저장 상태, 플랫폼 폼 선택, 내보내기 버튼

#### AI ↔ 원고 동기화 메커니즘 (핵심 설계)

- AI 응답 말미에 항상 **"원고에 추가"** 버튼 표시
- 버튼 클릭 시 삽입 위치 선택 팝업: `"커서 위치 삽입"` / `"챕터 말미 추가"` / `"새 단락 삽입"`
- 작가 선택 후 TipTap 에디터에 반영. 반영 즉시 버전 스냅샷 생성 (trigger: `ai_edit`)
- **AI가 원고를 자동 수정하는 경우 없음** — 반드시 작가 승인 후 반영

> ⚠️ **단일 기기 편집 원칙** 다중 기기 동시 편집은 Phase 1 미지원. 오프라인 편집 후 동기화 시 서버 버전과 충돌 감지 → 작가가 "서버 버전 유지" 또는 "내 편집 반영" 선택하는 팝업 표시.

#### 자동저장

- 30초마다 자동저장 (`chapter.content` → Supabase)
- 저장 상태: `"저장 중..."` → `"저장됨 HH:MM"` 상단 바 표시
- 네트워크 끊김 시 localStorage 임시 저장 → 재연결 시 동기화

### 4.2 AI 목차·제목 기획

- **입력** 책 주제 키워드, 예상 독자, 장르, 목표 분량
- **출력** 제목 후보 5개 + 목차(챕터명·소제목) + 예상 독자 프로필 + 추천 판매처
- 작가가 목차 확정 시 → `chapters` 테이블 자동 생성
- 이후 수동 편집 가능 (순서 드래그, 챕터 추가·삭제)

### 4.3 자료 검색 (Perplexity Sonar API)

#### 검색 플로우

1. 작가가 챗봇에 자연어로 검색 요청
2. Claude API가 검색 키워드 추출 → Perplexity Sonar API 호출
3. 결과에 신뢰도 도메인 필터 적용
4. 통과 결과 3~5개를 출처 링크와 함께 채팅창 표시
5. 작가가 "원고에 인용 추가" 버튼 → 각주 형태로 원고 삽입

#### 신뢰도 도메인 화이트리스트

| 분류 | 도메인 |
|------|--------|
| 국내 학술 | riss.kr, dbpia.co.kr, kiss.kstudy.com |
| 국내 뉴스 | yonhapnews.co.kr, chosun.com, joongang.co.kr, hani.co.kr, mk.co.kr |
| 국내 보고서 | kdi.re.kr, kisdi.re.kr, kostat.go.kr, motie.go.kr |
| 해외 학술 | pubmed.ncbi.nlm.nih.gov, arxiv.org, scholar.google.com, jstor.org |
| 해외 뉴스 | reuters.com, apnews.com, bbc.com, nytimes.com |

- 화이트리스트 외 도메인은 **"출처 확인 필요"** 배지 표시. 작가가 수동 변경 가능.
- 동일 검색어 24시간 내 캐시 재사용 (`search_results` 테이블). API 비용 절감.
- **횟수 차감 기준** API 호출 성공 + 결과 1건 이상 반환 시에만 차감. 실패·0건은 차감 없음.
- `count++` 는 결과 반환 직후에만 실행. 타임아웃·에러 catch 블록에서는 실행 안 함.

### 4.4 문체 교정 (역할 분리)

| 엔진 | 담당 역할 | 사용 시점 |
|------|-----------|-----------|
| 나라인포테크 API | 맞춤법·띄어쓰기·오탈자 교정 | 1순위. 계약 불발 시 → CLOVA Spell Checker 또는 Bareun.ai |
| Claude API | 문체 다듬기·어색한 문장 교열·말투 통일 | 문체 선택(전문체/구어체) 후 AI 요청 |

- **문체 옵션** 전문체(-이다/-한다) / 구어체(-이에요/-해요) / 중립체
- 교정 결과: 변경 전·후 비교 뷰 표시. 작가가 항목별 수락·거절 가능.

### 4.5 POD 워드 폼 생성 및 출력

| 플랫폼 | 판형 | 내지 폰트 | 여백 (상하/좌우) | 제출 형식 |
|--------|------|-----------|-----------------|-----------|
| 부크크 | A5 (148×210mm) | 나눔고딕 10pt | 20mm / 15mm | DOCX → PDF |
| 교보문고 POD | A5 / 신국판 | 나눔고딕 10pt | 플랫폼 지정 | DOCX |
| Amazon KDP | 6×9in (기본) | 임베딩 필수 | 0.75in / 0.5in+ | DOCX or PDF |

> **파일 생성 프로세스**
> 1. 작가가 플랫폼·판형 선택
> 2. 서버에서 해당 규격 DOCX 생성 (docx 라이브러리, 한글 폰트 임베딩)
> 3. 원고 내용 (챕터 순서대로) 삽입
> 4. 규격 검증 — 여백·폰트·페이지 사이즈 자동 체크
> 5. 검증 통과 시 다운로드. 실패 시 어떤 규격이 맞지 않는지 안내

> **Phase 1 범위** 텍스트 중심 원고만 완전 지원. 이미지 포함 원고는 DPI 체크(300DPI 미만 경고) + 블리드 확인 안내 제공. 이미지 원고 완전 지원은 Phase 2에서 고도화.

> ⚠️ 교보문고·부크크 템플릿 저작권: 플랫폼 문의 후 허가 확보 또는 규격만 재현한 자체 템플릿 사용.

### 4.6 버전 관리

- 스냅샷 생성 트리거: AI 수정 반영 (`ai_edit`) / 30초 자동저장 (`autosave`) / 수동 저장 (`manual`)
- 보관 수: 무료 챕터당 5개, 베이직 20개, 프로 무제한
- **롤백 UI** 우측 패널 "버전 히스토리" 탭 → 타임라인 리스트 → 미리보기 → 복원
- 복원 시 현재 버전을 먼저 스냅샷으로 저장 후 복원 (복원 실수 방지)

### 4.7 원고 내보내기

- 형식: DOCX / TXT / PDF — **전 플랜 무제한 제공**
- 챕터 선택 내보내기 가능 (전체 or 특정 챕터)
- `"언제든 원고를 꺼낼 수 있습니다"` 메시지 에디터 하단 상시 표시
- 서비스 해지 후 30일간 다운로드 유지

### 4.8 집필 진도 대시보드

- 목표 분량 (글자 수) 설정 → 전체·챕터별 진행률 바 표시
- 일별 집필량 막대 그래프 (최근 30일)
- **스트릭 카운터** 하루 100자 이상 작성 시 +1
  - 리셋 기준: **KST(한국 표준시) 자정(00:00)**
  - 100자 미만 시 스트릭 끊김. 당일 집필량 0으로 표시
  - **일시정지(Pause)** 월 1회 "오늘 쉬기" 버튼으로 스트릭 유지
- 예상 완성일 자동 계산 (최근 7일 평균 속도 기준)

## 7. 인증 및 결제 시스템

### 7.1 인증

- **방식** Supabase Auth — Google OAuth + 이메일/비밀번호
- 가입 즉시 `plan = free`, 세션 유지 30일
- 비회원: 무료 훅 (책 기획 체험·인세 계산기) 이용 가능. 저장 시 가입 유도
- 가입 완료 시 서버에서 `terms_agreed_at`·`terms_version`·`privacy_agreed_at` 자동 기록
- 약관 버전 관리: 약관 개정 시 기존 유저에게 재동의 요청 팝업 표시

### 7.2 구독 결제 (토스페이먼츠)

**최초 구독 플로우**
1. 업그레이드 버튼 → 플랜 선택 모달
2. 토스페이먼츠 결제창 (카드 정보 입력)
3. 빌링키 발급 → `subscriptions` 테이블 저장
4. `users.plan` 업데이트 → 기능 즉시 개방

**정기 결제**
- 매월 `next_billing_at` 기준 서버 크론 실행
- 결제 성공: `next_billing_at += 1개월`
- 결제 실패: 1·2·3일차 재시도. 최종 실패 시 `plan = free` 강등 + Resend 이메일 알림

**해지**
- 해지 즉시 `cancelled_at` 기록. 기간 만료까지 프로 기능 유지
- 만료 후 `plan = free`. 초과 프로젝트는 잠금 처리 (삭제 안 함)
- **잠금 상태 상세**
  - 원고 읽기·다운로드·내보내기는 가능
  - 편집·새 챕터 추가·AI 기능은 불가
  - 잠금 프로젝트는 목록에 표시되며 자물쇠 아이콘 + `"재구독하면 편집 가능"` 안내
  - 재구독 즉시 자동 잠금 해제 — 별도 복구 절차 없음

### 7.3 요금제

| 플랜 | 월납 | 연납 | 연납 절감 |
|------|------|------|-----------|
| 무료 | ₩0 | ₩0 | - |
| 베이직 | ₩9,900 | ₩99,000 | 2개월치 |
| 프로 | ₩19,900 | ₩199,000 | 2개월치 |

### 7.4 이메일 발송 (Resend)

- 무료 3,000건/월. 초과 시 $0.001/건
- **발송 도메인** 서비스 도메인 SPF·DKIM 레코드 설정 필수 (미설정 시 스팸 분류)
- **필수 템플릿 4종**
  1. 회원가입 인증
  2. 결제 성공 영수증
  3. 결제 실패 재시도 안내
  4. 구독 만료 D-7 알림

## 8. API 연동 명세

### 8.1 Claude API

| 용도 | 모델 | 주요 프롬프트 지침 |
|------|------|-------------------|
| 목차·제목 기획 | claude-sonnet-4-6 | 장르·독자·키워드 기반 상업적 제목 5개 + 챕터 구성 제안 |
| 본문 집필 보조 | claude-sonnet-4-6 | 작가 문체 유지, 기존 챕터 내용 참고, 표절 회피 지침 |
| 문체 교열 | claude-sonnet-4-6 | 전문체/구어체 선택값 적용, 원문 의미 훼손 금지 |
| 자료 요약 | claude-sonnet-4-6 | Perplexity 결과 300자 이내 요약, 출처 유지 |
| 번역 (한→영) | claude-sonnet-4-6 | 직역 아닌 문화적 현지화, 영미권 독자 기준 |
| 메타데이터 생성 | claude-sonnet-4-6 | KDP 정책 준수, SEO 최적화 키워드 7개 |

> 스트리밍 응답 필수 (`stream: true`). 모든 요청 `max_tokens: 4096`. 요청당 토큰 수 로깅으로 비용 모니터링.

### 8.2 Perplexity Sonar API

- **모델** `sonar` (경량·빠름, 출처 인용 내장)
- **엔드포인트** `POST https://api.perplexity.ai/chat/completions`
- `search_domain_filter` 파라미터로 화이트리스트 도메인 지정
- `return_citations: true` → 출처 URL·제목 자동 포함
- 캐싱: 동일 쿼리 24시간 내 `search_results` 테이블 재사용

### 8.3 맞춤법 교정 API (우선순위 순)

- **1순위** 나라인포테크 API — B2B 견적 계약. 개발 착수 전 계약 가능 여부 확인 필수
- **2순위 대안** 네이버 클라우드 CLOVA Spell Checker — 공개 가격, 즉시 API 키 발급 가능
- **3순위 대안** Bareun.ai 맞춤법 API — 스타트업 친화적 월정액, REST API 방식
- 공통 처리: 챕터 단위 500자씩 분할 전송. 결과: 오류 위치·교정 제안·오류 유형 배열
- 에디터 UI: 오류 구간 하이라이트 + 툴팁으로 교정안 표시. 항목별 수락·거절 가능

> ⚠️ 나라인포테크는 가격 미공개 B2B 방식. 스타트업 계약 거절 사례 있음. 반드시 사전 연락 후 대안 준비.

### 8.4 Resend — 이메일 발송

- 트랜잭셔널 이메일 전용 (마케팅 메일 아님)
- 무료 플랜: 3,000건/월
- 필수 템플릿 4종: 회원가입 인증 / 결제 성공 영수증 / 결제 실패 재시도 안내 / 구독 만료 D-7 알림

### 8.5 Rate Limiting — Upstash Redis

- **적용 레이어** Next.js API Routes 미들웨어에서 요청 차단
- **기능별 분당 제한**
  - 자료 검색: 3회
  - AI 집필 보조: 5회
  - 맞춤법 교정: 2회
  - 원고 진단: 1회
- **식별 기준** 로그인 사용자는 `user_id`, 비회원은 IP 주소
- **초과 시 응답** HTTP 429 + `"잠시 후 다시 시도해주세요 (N초 후 가능)"` 메시지

### 8.6 Inngest — 백그라운드 잡 큐

- Vercel API Routes 최대 60초 실행 한도 → 장기 작업은 Inngest로 위임
- **Inngest 처리 대상** 전체 원고 번역 / EPUB 생성 / 대용량 DOCX 생성 / 원고 진단 리포트 생성
- **플로우** 클라이언트 요청 → API Route가 Inngest 이벤트 발행 → 백그라운드 처리 → Supabase Realtime으로 완료 알림 → 클라이언트 UI 업데이트
- **처리 중 UI** `"번역 중... (약 2분 소요)"` 프로그레스 표시. 페이지 이탈해도 계속 처리

### 8.7 토스페이먼츠

- 빌링키 발급: `POST /v1/billing/authorizations/card`
- 정기 결제: `POST /v1/billing/{billingKey}`
- 웹훅: `payment.done` / `payment.failed` → 구독 상태 동기화

## 9. 랜딩 페이지 스펙 및 화면 흐름

> 에디터는 데스크톱 전용이나 랜딩 페이지 전체는 **모바일 반응형 필수**.

### 9.1 랜딩 페이지 섹션 구성

**섹션 1 — 히어로 (전환 핵심)**
- **헤드라인** `"AI와 대화하며 책을 씁니다. 기획부터 POD 출판까지."`
- **서브 카피** `"원고 집필 → 맞춤법 교정 → 판매 페이지 → 아마존 KDP까지. 작가에게 필요한 것만 담았습니다."`
- **CTA 주버튼** `"무료로 책 기획 시작하기"` → 비회원 기획 체험 인라인 입력창
- **CTA 부버튼** `"기존 원고 무료 진단받기"` → 파일 업로드 페이지
- **신뢰 배지** `"이 툴로 출판된 책 N권 · 작가 N명 이용 중"` (초기: 베타 참여자 수로 대체)

**섹션 2 — 인라인 체험 훅 (이탈 방어 핵심)**
- 랜딩 페이지 안에 주제 입력창 직접 삽입 — 다른 페이지로 이동 없이 결과 즉시 확인
- 입력: 책 주제 한 줄 → 출력: 제목 3개 + 목차 미리보기
- `"전체 결과 저장하기"` 클릭 시 → 회원가입 모달
- 인라인 인세 계산기 병행 배치: 페이지 수·판매가 입력 → 교보/KDP 예상 인세 즉시 계산

**섹션 3 — 핵심 기능 3가지**
- `"AI와 대화하며 집필"`: 자료 검색·본문 작성·교정을 채팅으로 지시
- `"POD 규격 자동 맞춤"`: 부크크·교보·KDP 규격 워드 파일 자동 생성
- `"셀링 페이지 AI 생성"`: 책 내용 기반 판매 문구·구성안 자동 제안

**섹션 4 — 사회적 증거 (신뢰 형성)**
- 이 툴로 출판된 책 표지 3~6개 수평 나열 (제목·저자명·판매 플랫폼 링크)
- 초기: 베타 테스터 10명 원고 완성 후 노출. 그전까지 섹션 숨김

**섹션 5 — 플랜 비교표 (결정 구간)**
- 무료 / 베이직 ₩9,900 / 프로 ₩19,900 카드 3개. 프로에 `"추천"` 배지
- 연간 할인 토글: 월납 ↔ 연납 전환 시 가격 실시간 변경
- **ROI 메시지** (플랜표 바로 위 고정): `"책 한 권 판매 수익 > 한 달 구독료. 팔리면 본전입니다."`
- **데이터 주권 안심 문구**: `"원고 저작권은 작가님께 있습니다. AI 학습에 사용하지 않습니다. 언제든 꺼내가실 수 있습니다."`

**섹션 6 — FAQ + 푸터**

|----------|-----------|

| POD가 뭔가요? | "주문 제작 출판. 책이 팔릴 때만 인쇄해 재고 위험이 없습니다." |
| 월정액 해지하면 원고 사라지나요? | "아니요. 해지 후에도 읽기·다운로드 가능. 재구독하면 편집도 복원." |

### 9.2 신규 작가 온보딩 플로우

1. 랜딩 페이지 히어로 섹션 → `"무료로 책 기획 시작하기"` 클릭
2. 인라인 체험: 주제 입력 → 제목+목차 생성 → `"저장"` 시 가입 모달
3. Google OAuth 또는 이메일 가입
4. 가입 직후: 진단 이력 있으면 `"방금 진단한 원고가 계정에 연결됐습니다"` 안내
5. 에디터 진입 → 집필 시작

### 9.3 집필 → 출력 전체 플로우

1. 프로젝트 생성 → 플랫폼 + 판형 선택
2. 목차 기획 (AI 제안 or 직접 입력)
3. 챕터별 집필 (에디터 좌측) + AI 챗봇 활용 (우측)
4. 자료 검색 → 인용 삽입
5. 맞춤법 교정 (나라인포테크 API) → 문체 교열 (Claude)
6. 규격 검증 → POD 워드 폼 다운로드
7. (프로) 셀링 페이지 생성 → 캔바 편집
8. (프로) KDP 글로벌: 번역 → EPUB 생성 → 제출 패키지 ZIP 다운로드

### 9.4 유료 전환 트리거 포인트

| 상황 | UI 처리 |
|------|---------|
| 자료 검색 월 한도 초과 | `"이번 달 검색 횟수를 모두 사용했습니다"` + 베이직 업그레이드 모달 |

### 11.3 환경변수 관리 전략

- 모든 API 키는 **Vercel 환경변수에만 저장** — 코드·git에 하드코딩 절대 금지
- 로컬 개발용 `.env.local` 파일은 `.gitignore`에 반드시 포함
- **키 접두어 규칙** 서버 전용 `SECRET_` / 클라이언트 공개 `NEXT_PUBLIC_`

| 환경변수명 | 용도 |
|-----------|------|
| `SECRET_ANTHROPIC_API_KEY` | Claude API 호출 (서버 전용) |
| `SECRET_PERPLEXITY_API_KEY` | Perplexity Sonar API 호출 (서버 전용) |
| `SECRET_TOSS_SECRET_KEY` | 토스페이먼츠 결제 (서버 전용) |
| `SECRET_RESEND_API_KEY` | 이메일 발송 (서버 전용) |
| `SECRET_INNGEST_SIGNING_KEY` | Inngest 백그라운드 잡 인증 (서버 전용) |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL (클라이언트 공개) |
| `SECRET_SUPABASE_SERVICE_KEY` | Supabase 서버 어드민 키 (서버 전용) |
| `SECRET_UPSTASH_REDIS_URL` | Rate Limiting Redis 연결 (서버 전용) |

> ⚠️ GitHub Actions·CI/CD 환경에서도 Vercel 환경변수를 직접 참조. 절대 secrets를 코드 레포에 커밋하지 말 것.
